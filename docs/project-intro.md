# 项目介绍（手搓版 Token SaaS V1）

## 概述

这是一个**参考 New API 的产品结构与主链思路**、但**不照搬其现成底座实现**的手搓版 Token SaaS V1。

目标是尽快跑通最小闭环，同时保证：

- **控制权在自己手里**
- **可追踪、可定位**
- **可替换、可演进**
- **降低“黑盒权限/风控/泄露责任不清”的风险**

V1 主链：

```text
用户登录
→ 看到套餐/状态
→ 发放平台 token
→ token 调你自己的网关
→ 你自己的网关转发上游
→ 返回模型结果
→ 写最小日志
```

## 核心能力（V1 范围）

- **统一接口**：网关对外提供 OpenAI-compatible 形态（如 `/v1/models`、`/v1/chat/completions`）
- **最小权限**：token 启用/停用 + 用户状态（active）+ 模型白名单
- **最小审计**：请求成功/失败/拒绝都落日志（`usage_logs`）
- **可观测性**：
  - Web：可选 GA4 / Umami（环境变量启用，零改页面内容）
  - Gateway：可选 pprof 风格端点 / Pyroscope（环境变量启用，零代码集成）

> 说明：V1 **刻意不做**分销/复杂计费/复杂风控/多租户组织权限/白标等平台化能力，避免范围失控。

## 与 New API 的关系（取其精华，去其糟粕）

- 我们**借鉴**：产品链路、统一接口形态、token 概念、最小审计思路
- 我们**不继承**：其现成后台/权限/计费/多租户等复杂实现与黑盒逻辑

参考信息见：`docs/new-api-reference.md`
